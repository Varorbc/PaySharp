测试卡号：与银联测试环境联调使用的卡号 测试卡号

注：因配样例中测试商户号777290058110097银联后台配置的是 借记卡是 前台开通的权限，贷记卡是后台开通权限，故本“无跳转token-银联侧开通”样例只能使用借记卡号测试。

交易流程：
获取token号：前台开通交易,开通查询（根据前台开通的订单号orderId查询获取token)
消费： 使用token号做消费短信交易(如果商户号配置验证短信), 使用token号做消费交易
后续可以对消费交易做退货，消费撤销交易
后续可以对token号做解除标记交易
可以对消费，退货，消费撤销进行交易状态查询交易
可以通过对账文件下载交易获取商户的对账文件

前台开通：有后台通知，有前台返回商户按钮（前台通知）
通过后台通知获取token号

开通查询：后台交易，无后台通知
如果前台开通没有收到后台通知无法获取token号，可以调用次交易获取token号

消费短信：后台交易，无后台通知
上送手机号，金额，token号等发送消费短信

消费：后台交易，有后台通知
上送短信验证码，token号做消费交易

解除标记：后台交易，无后台通知
解除token号。

交易状态查询说明：
origrespcode=00成功，03、04、05重新查询，其他为失败。

消费撤销交易与退货：
消费撤销和退货有什么区别？
消费撤销仅能对当天的消费做，必须为全额，一般当日或第二日到账，可能存在极少数银行不支持。
退货能对11个月内的消费做（包括当天），支持部分退货或全额退货，到账时间较长，一般1-10天（多数发卡行5天内，但工行可能会10天），所有银行都支持。
注：以上的天均指清算日，一般前一日23点至当天23点为一个清算日。测试环境为测试需要，13:30左右日切，所以13:30到13:30为一个清算日。

对账文件下载：
对账文件什么时候能下载？
测试环境一般下午5点出，文件内包含的交易的时间范围是13:30-13:30。
生产环境一般早上9点出，文件内包含的交易的时间范围是23:00-23:00。

对账文件获取后会落地成一个zip文件，zip文件中的ZM，ZME文件各个字段的拆分解析可以参考DemoBase.java中的parseZMFile parseZMEFile 方法。
